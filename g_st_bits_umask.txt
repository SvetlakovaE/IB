Установка группового бита (на файлы и на папки)
sudo chmod g+s имя объекта
sudo chmod g+s script.bash
установка g-бита на файл позволяет запускать файл от имени группы
sudo chmod g+s fold1
установка g-бита на файл позволяет наследовать группу владельцев на все подпапки, раположенные внутри
Смена группы владельцев у папки Fold1
sudo chown :group1 fold1
mkdir fold1/fold_test создает папку с группой владельцев group1 и наследует g-бит от родительской папки 

пример прав доступа к файлу с g-битом
2664/rw-rwSr--


пример прав доступа к папке с g-битом
2775/rwxrwsr-x

установка sticky-бита на папки
sudo cmod out имя папки
по умолчанию файлы, созданные владельцем внутри папки, могу удалять либо владельцы, либо члены группы владельцев
папки с установленным t-битом запрещают удаление файлов всем кроме владельцев. 
при этом при разрешении на запись можно удалить любую информацию находящуюся внутри такого файла.
sudo chmod o+t fold3
touch fold3/test_tbit.txt
stat fold3/test_rbit.txt
0664/rwx-rw-r-- uid student/1000 GID group1/1001
newpp group1
cat> test_tbit.txt
файл доступен для записи
rm test_tbit.txt
сообщение от команды rm:

пример доступа к папке c t-битом
1775/rwxrwxr-t

при одновременной установке g и t битов на папку наследование распространяется только на g бит
